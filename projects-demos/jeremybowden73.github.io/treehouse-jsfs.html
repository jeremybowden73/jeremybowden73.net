Pagination and Content Filter


Using native JavaScript I added pagination to a webpage displaying 54 students, so only 10 students would be displayed per page. This has the benefit to the user of making the list more manageable and easier to read. It required JavaScript code be written to split the list into groups of 10 students, display each group as a page, and add pagination buttons to the bottom of the page to enable a user to jump straight to a specific page.


I added a text-entry keyword search field to the page, to search on name or email address. The results are then paginated in the same way as the original list. Finally, if the search yielded no results, I included a (rather crude!) message to inform the user, which then automatically clears and returns the user to the main list after 5 seconds.


This project helped to reinforce my knowledge of event listeners, event handlers, traversing the DOM with JavaScript, and injecting code into the HTML with JavaScript.

Interactive Form with Validation


This project is all about using client-side JavaScript to dynamically update HTML form elements (like text entry, dropdowns, checkboxes, and submit elements) for a conference application form, and validate data when the form is submitted by the user.


Using only native JavaScript I created the necessary code to check for acceptable email addresses, filter options depending on another selection, show and hide browser elements when required, and provide error messages if invalid data is entered. All of this is done when the user clicks the Submit button.

Employee Directory


Here, I created an employee directory, using the API of "randomuser.me" to get randomly-generated employee profiles. I used the new native JavaScript method Fetch to do this, rather than a jQuery AJAX method or older JavaScript options.


Once I'd got a batch of 12 profiles I used CSS's Grid layout to display them by creating a smart UI that is fully responsiveness regardless of the device or screen size that the user is viewing the webpage on. When a profile is clicked, a modal window pops up to show full details for the employee.


This project developed my understaning of using website APIs, using Fetch for AJAX requests, processing data returned from the server, and consolidated my knowledge of DOM manipulation and event handlers in JavaScript.

Webpage Content Scraper using Node.js


Wow! This is different!

My first foray into programming JavaScript using Node.js. This command-line application is a web scraper that harvests data from an E-commerce website, using npm modules like "req" to make http requests and "cheerio" to build the scraper.


When the home page of the E-commerce site is visited, the program accesses every product page within the site that represents an item of clothing. It then gets, and saves to a CSV file, relevant information for the item (e.g. price, title, image URL). If the http request returns an error, the errors are logged to a mutable log file.


Whilst working on this project I learned a lot about server-side JavaScript, using npm modules in Node, and manipulating files in the local file system. It was a bit of a head-scratcher to begin with but the puzzle pieces slotted together after a little whle!
